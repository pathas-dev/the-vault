# Routes Agent Guidelines

Route-specific conventions and architectural patterns for `/src/routes/`.

## Routes Overview

| Route         | Purpose                                                                                                                                  | Key Components                              |
| ------------- | ---------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------- |
| `__root.tsx`  | Root layout wrapper. Applies dark theme pre-render, TanStack devtools.                                                                   | `RootDocument`, theme init script           |
| `index.tsx`   | Home/landing page. Hero section with CTA buttons, operation overview cards.                                                              | `HomeScreen`, session detection             |
| `record.tsx`  | Main feature (~620 lines). Multi-round vault recording form with target house, floor plan grid, wall/entry selection, vault value input. | `RecordScreen`, `WallMiniMap`, keyboard nav |
| `summary.tsx` | Results page. Total extraction value, per-round breakdown (desktop table + mobile cards), `WallMiniMap` visualization.                   | `SummaryScreen`, calculation helpers        |
| `about.tsx`   | About page. Static content.                                                                                                              | `About`                                     |

## Route Definition Pattern

All routes use `createFileRoute` from TanStack Router with consistent export:

```typescript
export const Route = createFileRoute('/path' as any)({
  component: ScreenName,
})
```

Screen components are regular function components (not separately exported). Type annotation `as any` is required.

## Session Data Management

Session data stored in `sessionStorage` under key `vault_rounds` as JSON array.

**RoundData type:**

```typescript
{
  targetHouse: string // 'A' | 'B' | 'C' | 'D'
  startPoint: string // 'A' | 'B'
  horizontalWall: string | null // 'ㄴ' | 'ㄷ' | null
  verticalWall: string | null // 'a' | 'b' | 'c' | 'd' | null
  vaultValues: Record<string, string[]> // vault number -> [values]
}
```

**DO NOT** change the key name `vault_rounds`.

## Styling Rules

- **Page wrapper:** `flex-1 flex flex-col` with `pt-16 md:pt-20` for Topbar clearance
- **Mobile-first:** Base styles are mobile; use `md:` breakpoint for desktop
- **Fixed bottom actions (mobile):** `md:hidden fixed bottom-6` for button groups
- **Desktop actions:** `hidden md:flex` with centered layout and glow effects
- **Background texture:** External image URL at very low opacity (`opacity-[0.04]`)

## Shared Components

- `Topbar` — Layout header (import from `../components/Layout`)
- `WallMiniMap` — Floor plan visualization (import from `../components/WallMiniMap`)

## Keyboard Navigation

`record.tsx` implements Enter/Arrow key navigation for vault value inputs:

- **Enter:** Move to next input (down column, then next vault)
- **ArrowDown:** Jump to next vault at same column index
- **ArrowUp:** Jump to previous vault at same column index

## Important Constraints

- **DO NOT** edit `routeTree.gen.ts` — auto-generated by TanStack
- **DO NOT** change `sessionStorage` key name
- **DO NOT** add routes without running dev server (TanStack auto-generates route tree)
- Routes are **not nested** — all top-level routes map to their own screens
